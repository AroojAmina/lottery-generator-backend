🚀 HOW TO USE
========================================================================

STEP 1: Prepare Input File
   • File name: numbers.txt
   • Format: 07,16,26,29,52,04 (one line per ticket)
   • Numbers: 5 main (1-60) + 1 cash ball (1-4)
   • Place in same folder as LotteryPrinter.exe

STEP 2: Run Program
   • Double-click: LotteryPrinter.exe
   • Program automatically:
     ✓ Reads numbers.txt
     ✓ Validates all numbers
     ✓ Creates printed_slips.pdf
     ✓ Sends to printer
     ✓ Shows confirmation message

STEP 3: Print (if auto-print doesn't work)
   • Open: printed_slips.pdf
   • Right-click → Print (or Ctrl+P)
   • Print in COLOR (important for red cutting lines)

🔧 CUSTOMIZATION GUIDE
========================================================================

To adjust appearance, edit lottery_printer\config.py:

Fine-tune alignment:
   X_START = 100.0          ← Shift grid left/right
   Y_TOP_ANCHOR = 190.0     ← Shift grid up/down

Change dot size:
   CIRCLE_RADIUS = 3.8      ← Larger = bigger dots

Adjust spacing:
   COL_WIDTH = 10.5         ← Space between columns
   ROW_HEIGHT = 10.2        ← Space between rows
   GAME_SPACING = 78.0      ← Space between 5 lines

After editing, rebuild .exe:
   cd "d:\Lottery project"
   env\Scripts\python -m PyInstaller --onefile --windowed --name=LotteryPrinter lottery_printer\main.py

📞 SUPPORT & DEBUGGING
========================================================================

Error: "No .txt file found"
   → Ensure numbers.txt is in same folder as .exe
   → Check file name is exactly "numbers.txt"

Error: "Invalid numbers"
   → Format: 07,16,26,29,52,04 (no spaces)
   → Main numbers: 1-60
   → Cash ball: 1-4

PDF not printing:
   → Check default printer is set in Windows
   → PDF file is still created even if print fails
   → Can print manually from printed_slips.pdf

Numbers look misaligned:
   → Edit config.py values:
      - X_START (horizontal shift)
      - Y_TOP_ANCHOR (vertical shift)
      - COL_WIDTH / ROW_HEIGHT (grid spacing)
   → Rebuild .exe after changes
